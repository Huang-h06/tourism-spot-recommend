# 个人实训总结报告

## 项目概述

本项目是一个全栈"旅游景点推荐系统"，前端基于原生 HTML/CSS/JS，后端采用 Flask + Supabase 云数据库，实现景点浏览、用户注册登录、评分评论、游记分享、地图定位和管理员后台等完整功能。项目拥有 5 个独立页面路由、44 个完整 API 接口，代码托管于 GitHub 公开仓库，在 6 个不同日期提交了 20+ 次有效 commit。

## 遇到的最大困难及解决过程

开发过程中遇到的最大困难是**管理员权限判断失败**。

我按照 AI 给出的 SQL 成功将用户设为管理员后，登录 admin 账号访问管理后台时提示"无管理员权限"，无法进入。一开始以为是数据库 SQL 没执行成功，反复检查了 migrate.sql 的执行结果、确认 users 表已经有 role 字段且值为 'admin'，但问题依旧。

经过排查发现：`db_supabase.py` 中的 `get_user_by_id()` 函数在查询用户时只取了 `id, username, email, avatar_url, created_at`，没有查询 `role` 字段。所以后端 API `/api/admin/check` 调用该函数时拿不到 role，`user.get("role") == "admin"` 永远是 False。

修复方案分两步：在 `get_user_by_id()` 的 SELECT 语句中加入 `role, enabled`；在登录接口的返回值中加入 `role` 字段让前端 localStorage 也同步角色信息。

这个过程让我深刻体会到：看似简单的一个字段遗漏，会导致整个权限系统失效。排查时要从 API 返回值逐层倒推，才能定位根因。

另一个困难是地图底图不显示。Leaflet 默认使用 OpenStreetMap 瓦片，但该服务在国内被墙。一开始以为是 Leaflet 或浏览器兼容性问题，后来定位到是网络层面的 CDN 被墙。解决方案是将瓦片源更换为高德地图，使用国内可访问的 `webrd0{s}.is.autonavi.com`，问题解决。

## AI 辅助编程的心得体会

本次实训大量使用 AI（CodeBuddy）进行辅助开发，整体体验如下：

### 优点

1. **效率提升显著**：以前写一个游记模块（数据库表 + API + 前端页面）可能要半天，AI 在几分钟内就能生成完整可用的代码框架，我只需要微调和集成
2. **排查问题快速**：之前需要逐行查日志、搜 StackOverflow 的问题，现在直接截图或描述症状给 AI，它能快速定位根因并给出修复方案
3. **降低技术门槛**：我之前对 Leaflet 地图、Supabase RLS 策略等不熟悉，但通过 AI 的引导能快速上手

### 缺点 / 注意事项

1. **AI 不知道你的数据库实际状态**：它生成的 SQL 用的字段名可能与实际不一致（如 `password_hash` vs `password`），需要自己核实
2. **AI 不了解国内网络环境**：默认用 OpenStreetMap 瓦片，对墙内用户不友好，需要人工指出问题
3. **大文件编辑风险**：AI 修改大段代码时可能出现遗漏，要仔细对比 diff

### 核心原则

**AI 是副驾驶，人必须当主驾驶。** 代码生成后要逐行 review，运行前要确认依赖版本、数据库表结构、网络环境是否匹配。Prompt 写得越具体（附上文件路径、截图、报错信息），AI 的输出越准确。

## 个人在项目中的具体贡献

1. **数据库设计**：设计了 7 张业务表（spots、users、comments、favorites、browsing_history、videos、travel_notes），编写了完整的迁移 SQL
2. **后端开发**：编写了 `app.py`（44 个路由）和 `db_supabase.py`（30+ 个数据库操作函数），实现了用户认证、评分评论、收藏、浏览记录、游记 CRUD、管理员后台全套 API
3. **前端开发**：编写了 5 个独立页面模板（首页、详情页、个人主页、游记广场、管理后台），实现了轮播图动态加载、评分互动、地图组件、美食画廊等前端交互
4. **素材整理**：收集了 12 个城市的景点封面图、美食图片和宣传视频素材
5. **文档撰写**：编写了完整的 README、API 文档、Prompt 日志和个人总结报告
6. **Git 管理**：在 6 个不同日期提交了 20+ 次有意义的 commit，包含清晰的 feat/fix docs 标注

---

## 项目存在的不足与改进方向

1. **未实现 CI/CD**：目前依赖手动启动，未来可接入 GitHub Actions 自动部署到 Render 等平台
2. **缺少单元测试**：API 接口应补充 PyTest 自动化测试
3. **图片上传**：游记图片目前只支持 URL 粘贴，后续可接入文件上传
4. **前端框架**：当前是原生 JS，后续可用 Vue/React 重构提升可维护性

## 总结

通过本次实训，我完整实践了一个全栈 Web 项目从 0 到 1 的开发流程：需求分析 → 数据库设计 → 后端 API → 前端页面 → 测试调试 → 文档撰写。同时深刻体验了 AI 辅助编程的工作模式，学会如何更高效地与 AI 协作完成复杂任务。这些经验对未来从事软件开发工作具有重要价值。
